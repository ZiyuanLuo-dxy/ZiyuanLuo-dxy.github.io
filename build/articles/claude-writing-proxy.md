# 用 Claude 中转做写作与文案：接入与实用技巧

我是个靠码字吃饭的人——公众号、品牌文案、产品软文都写。这两年我最顺手的工具就是 Claude，但一开始被"怎么稳定用上"卡了很久。折腾了大半年，踩了不少坑，这篇就把我作为一个"非程序员写作党"的经验掰开讲清楚：为什么选它、怎么接、怎么把稿子写得又快又好、以及怎么少花钱。

## 一、为什么写作的人都爱用 Claude

先说结论：在我用过的一堆模型里，Claude 是最像"一个有文字审美的人"在跟你合作，而不是那种堆词的机器。具体到我日常写作，有三点特别打动我。

**文笔自然，不端着。** 很多模型写出来的东西一眼假：排比句一串、"赋能""抓手"满天飞、每段结尾都要升华一下。Claude 相对克制，你让它写口语它就真口语,让它模仿某种调性它能压住那股 AI 腔。对文案人来说,这意味着改稿工作量小很多。

**长文不散架。** 我经常一次性喂进去几千字的资料,让它写一篇两三千字的深度稿。Claude 的上下文很长,前面交代的设定、语气、禁忌词,写到最后一段它还记得住,不会写着写着人设跑了、口径变了。这点在写系列稿、长专题时太重要。

**理解力强,能听懂"言外之意"。** 你说"这段太硬了,软一点",它知道你要的是减少说教、加点人味,而不是简单换几个形容词。你给它一堆零散笔记,它能理出逻辑、补上过渡。它更像个能对话的搭档,而不是一个只会执行指令的填词器。

当然它也不是万能的,时效性强的事实、精确数字还得自己核。但作为"初稿引擎 + 改稿搭档",它目前是我心里的第一梯队。

## 二、不折腾梯子的接入方式

写作党最怕的就是技术门槛。官方直连要美区信用卡、要稳定的境外网络,账号还时不时被风控,充了钱用不上非常闹心。我后来改用国内中转,思路很简单:**不改变你熟悉的软件,只是把它连接的"地址"换一下**。

我现在主用 KingFlow 的中转,下面三种接法你按自己习惯选。

**方式一:桌面客户端(最推荐给非程序员)**

装一个 GUI 客户端就行,比如 ChatBox、Cherry Studio、NextChat、OpenCat 这类。进设置,把 API 提供方选成 Anthropic 兼容,然后填三样东西:

- Base URL(中转地址):`https://www.kingflow.ai/v1`
- API Key:在 KingFlow 后台生成的那串
- 模型:写作我常用 `claude-opus-4-8`(精修)或 `claude-sonnet-4-6`(初稿)

填完保存,界面跟平时一模一样,只是背后走的是中转,国内直接能连,不用挂任何代理。

**方式二:网页直接聊**

懒得装软件的话,很多兼容的网页端也支持填中转地址和 Key,打开浏览器就能用,适合临时改两句文案的场景。

**方式三:API(给会一点代码的朋友)**

如果你想批量跑稿、接自己的小工具,用官方 SDK 只需改两个环境变量:

```bash
export ANTHROPIC_BASE_URL="https://www.kingflow.ai"
export ANTHROPIC_AUTH_TOKEN="你的_KingFlow_Key"
```

然后正常调用 `/v1/messages` 就行,一行 Base URL 的事,原来的代码基本不用动。

## 三、写作实用技巧:让它真的替你干活

工具接好只是开始,真正拉开差距的是怎么用。分享几个我天天在用的套路。

**1. 先给角色设定,再让它写。** 不要上来就说"帮我写篇稿"。我一般先铺一段:"你是一个写生活方式类内容的资深编辑,语气亲切、爱用具体细节和小故事,不用空洞的大词。目标读者是 25-35 岁的都市女性。"把人设、语气、读者画像一次交代清楚,后面所有产出都会稳定在这个调性上,省去反复纠正。

**2. 大纲先行,再逐段扩写。** 直接让它写全文,容易结构松垮。我的做法是分两步:先让它给我三个不同角度的大纲,我挑一个、改几处;然后按大纲一段一段扩写。这样我对文章骨架有掌控,它负责填肉,配合最顺。

**3. 改稿靠迭代,别指望一遍到位。** 初稿出来后我会给很具体的反馈:"第二段开头太平,加个反常识的钩子""这里说教味重,改成讲一个场景"。它改得很快,你也能看着稿子一版版逼近想要的样子。把它当成一个不知疲倦、还不会情绪化的编辑,反复磨就对了。

**4. 建个"风格样本"。** 把你过往写得最满意的两三段贴给它,让它照着这个感觉写。有了参照物,它的输出会明显更贴近你的个人风格,而不是那种没有辨识度的通用腔。

## 四、怎么省钱:好钢用在刀刃上

用得多了,成本自然要算。写作场景其实很容易省,关键是**分档使用**。

**初稿用便宜的,精修用旗舰的。** 出大纲、铺初稿、批量生成候选标题这种"量大但不那么讲究"的活,我交给 `claude-sonnet-4-6`,高频、批量的小活甚至用 `claude-haiku-4-5`,又快又省。等到要定稿、抠语气、打磨金句这种关键环节,再换 `claude-opus-4-8` 上场精修。一篇稿子下来,大头的 token 都花在便宜档,成本能压下来不少。

**复用风格设定,别每次重发。** 前面说的那套角色设定、风格样本,如果每次对话都完整重发,是笔不小的开销。KingFlow 完整透传了 Prompt Cache,把这些固定的前置内容缓存下来,后续调用重复读取时的成本会明显下降。对写作这种"设定长、正文相对短"的场景特别划算——你只要在同一套设定下连着写,缓存的性价比就出来了。具体能省多少看你的用法,我这边实测重复部分的费用确实降了一大截。

**一个 Key 切着用。** 因为一个 Key 就能调所有模型,我在初稿和精修之间切换只是改个 model 参数,不用维护好几套账号密钥,心智负担小很多。

## 五、为什么我用 KingFlow

写作这行,稳定和省心比什么都重要。用中转前我试过自己挂代理连官方,高频长连接跑着跑着就断,一断思路就没了,非常影响状态。后来选 KingFlow,主要看中这几点:

- **国内直连,响应快。** 走国内节点,首字出来通常一两秒,不用自己折腾梯子,打开就能写。
- **人民币小额充值。** 微信支付宝直接付,新人注册一般还送点额度,可以先测顺手了再充,不用一上来砸钱。
- **一个 Key 多模型。** Claude 三档随便切,偶尔想换别的模型对比一下文风也行,一套 Key 全搞定。
- **用量透明可对账。** 后台能看日志、余额、每次调用花了多少 token,月底算成本清清楚楚,不会稀里糊涂扣钱。

对我这种非技术出身、只想安心码字的人来说,这套组合基本做到了"接上就忘了它的存在",这恰恰是好工具该有的样子。

## 六、FAQ

**Q1:我完全不懂代码,能用起来吗?**
能。装一个 ChatBox 或 Cherry Studio,在设置里填 Base URL(`https://www.kingflow.ai/v1`)、API Key 和模型三样东西就行,全程点鼠标,不用写一行代码。

**Q2:用中转会不会影响 Claude 的写作质量?**
不会。中转只是换了连接的地址,背后调用的还是同一个模型。你选 `claude-opus-4-8`,产出的就是旗舰的水准,文笔、理解力和直连没区别。

**Q3:初稿和精修分别该选哪个模型?**
初稿、出大纲、批量生成这类活选 `claude-sonnet-4-6`,高频小活可以用 `claude-haiku-4-5`;定稿、抠语气、打磨关键段落用 `claude-opus-4-8`。分档用最省钱。

**Q4:我担心充了钱用不上,能先试试吗?**
可以。KingFlow 一般对新人有赠送额度,支持人民币小额充值,建议先用送的额度跑几篇稿,顺手了再按自己的量充,风险很低。
